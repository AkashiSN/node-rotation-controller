node-rotation-controller {{ .Chart.AppVersion }} is installed in namespace {{ .Release.Namespace }}.

It rotates Karpenter-managed nodes make-before-break within your maintenance
window, before Karpenter's forceful expireAfter fires (spec §3).

Rotation policy is a cluster-scoped RotationPolicy CRD (spec §5.4) — one
object per divergent policy.

Next steps:

{{- if .Values.rotationPolicies }}

1. RotationPolicy objects were installed from rotationPolicies:
{{- range .Values.rotationPolicies }}
{{- if or (not (hasKey . "create")) .create }}
     - {{ .name | default (include "node-rotation-controller.fullname" $) }} -> nodePoolSelector {{ .spec.nodePoolSelector }}
{{- end }}
{{- end }}
   Each governs the NodePools its nodePoolSelector matches. Point one at your
   NodePools by giving it a nodePoolSelector that matches them and a window you
   are willing to be disrupted in. Helm replaces list values whole, so pass the
   entry entire (or use -f values.yaml):

     helm upgrade {{ .Release.Name }} <chart> \
       --namespace {{ .Release.Namespace }} \
       --set-json 'rotationPolicies=[{"spec":{"nodePoolSelector":{"matchLabels":{"workload":"api"}},"maintenanceWindows":[{"timezone":"Asia/Tokyo","days":["Wed","Sat"],"start":"02:00","end":"06:00"}]}}]'

   A NodePool matched by no RotationPolicy is not rotated (its expireAfter
   backstop still applies); an equal-specificity tie between policies is a hard
   PolicyConflict (spec §5.4).
{{- else }}

1. rotationPolicies is empty — no RotationPolicy was installed. Author one
   per NodePool group you want rotated (spec §5.4):

     kubectl apply -f - <<'EOF'
     apiVersion: noderotation.io/v1alpha1
     kind: RotationPolicy
     metadata:
       name: api
     spec:
       nodePoolSelector:
         matchLabels:
           workload: api
       maintenanceWindows:
         - timezone: Asia/Tokyo
           days: [Wed, Sat]
           start: "02:00"
           end: "06:00"
     EOF

   A NodePool matched by no RotationPolicy is not rotated (its expireAfter
   backstop still applies).
{{- end }}

3. Confirm the controller is healthy and which replica holds the leader Lease:

     kubectl -n {{ .Release.Namespace }} get deploy {{ include "node-rotation-controller.fullname" . }}
     kubectl -n {{ .Release.Namespace }} get lease node-rotation-controller.noderotation.io
     kubectl get rotationpolicies

Prerequisite: Karpenter v1+ must already be installed (this chart does NOT
install Karpenter or its CRDs). The controller never bypasses Karpenter — it
induces replacement capacity via a placeholder Pod and deletes old NodeClaims.
