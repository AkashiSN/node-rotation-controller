# Documentation rules bootstrap

Before creating or modifying project documentation:

1. Read `docs/development/documentation-style.md` in full.
2. Apply its authority, document-profile, translation, safe-change, and
   validation rules.
3. Treat that file as the single source of truth. This steering file only
   selects when the shared rules apply.

These rules protect specification accuracy, EN/JA synchronization, operational
safety, and consistent output from humans and different AI agents.
